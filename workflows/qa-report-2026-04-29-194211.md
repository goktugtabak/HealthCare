## Findings by Severity

### CRITICAL (3)

**C-01 — Rate-limit bypass via X-Forwarded-For** _(adversarial)_
NFR-07 (10 fail / 15 min) is fully defeated. Unlimited brute-force on `/api/auth/login` and `/api/auth/register`, plus the global 100 req/min cap, is bypassed by spoofing the `X-Forwarded-For` header. Express trusts the header by default in dev. Fix: configure `app.set('trust proxy', ...)` to a specific value (number of hops or specific IPs), and ensure `express-rate-limit`'s `keyGenerator` uses the resolved client IP, not the raw header.

**C-02 — Admin can self-hard-delete (no second-admin / no safety check)** _(adversarial)_
Admin (the only admin in seed data) can call `POST /api/admin/users/:self/hard-delete` and instantly brick the platform with no recovery path through the API. Fix: refuse hard-delete on the calling admin's own ID; refuse if it would leave zero admins.

**C-03 — Admin can self-suspend; cannot reactivate self afterwards** _(adversarial)_
Same shape as C-02. Suspending the only admin removes platform recovery via UI/API. Fix: same — refuse self-suspend; require ≥1 active admin invariant.

### HIGH (10)

**H-01 — Confidentiality is not enforced on post reads** _(adversarial)_
Any authenticated user (engineer, healthcare, anybody verified) can `GET /api/posts/:id` of a `confidential` or `highly_confidential` post owned by anyone else. `listPosts` and `getPost` have no confidentiality filter — the schema's three-level field is purely cosmetic.

**H-02 — Frontend route guard is JS-only (clientside)** _(adversarial)_
With JS disabled, direct URL nav to `/admin/*` exposes the bare HTML shell. Backend correctly enforces RBAC on the API, but the SPA doesn't return a server-redirect — fine if backend stays the source of truth, but a leak of admin route names + structure.

**H-03 — Email enumeration via response timing** _(adversarial)_
Existing email + wrong password takes detectably longer than non-existent email (bcrypt vs. early-return). Fix: dummy bcrypt compare on the not-found branch.

**H-04 — Stack traces leaked in error responses** _(adversarial)_
Some 500 paths return raw Node stack traces. Fix: scrub error.stack in production responses; ensure `NODE_ENV=production` masks them.

**H-05 — No per-user write rate limit on `POST /api/posts`** _(adversarial)_
Created ~150 spam posts in seconds during the audit. No per-user cap.

**H-06 — No per-user rate limit on meeting requests / notification bombing** _(adversarial)_
Same shape as H-05 for meetings and notifications.

**H-07 — NDA "satisfied" check accepts cancelled/declined meeting requests** _(adversarial)_
The NDA gate logic considers any meeting request (regardless of status) as satisfying the requirement — declined / cancelled rows count.

**H-08 — Forms: 17 screens have unlabeled inputs** _(ux)_
`/login`, `/register`, `/profile`, `/create-post`, `/edit-post/p1`, search bars across `/explore`, `/my-posts`, `/admin/users` — placeholder-only, no `<label for>` / `aria-label` / wrapping `<label>`. Screen-reader hostile.

**H-09 — Buttons: 17 screens have icon-only buttons with no `aria-label`** _(ux)_
Close, search, menu, like icons all rely on tooltip-on-hover; fail SR.

**H-10 — Contrast: 39 text nodes likely fail WCAG AA (4.5:1)** _(ux)_
Concentrated on `/`, `/privacy`, `/terms` (gradient/glass surfaces and de-emphasized legal copy).

### MEDIUM (6)

**M-01 — Refresh tokens not rotated on use** _(adversarial)_ — `/api/auth/refresh` returns a new access token but the same refresh token; rotation is FR-recommended for replay defense.
**M-02 — Path-traversal-with-null-byte triggers 500** _(adversarial)_ — info leak / DoS surface on certain routes.
**M-03 — XSS payloads stored verbatim** _(adversarial)_ — server doesn't sanitize; relies on React escaping. Any future `dangerouslySetInnerHTML` introduces XSS instantly.
**M-04 — Email-export of pending-deletion users still works** _(adversarial)_ — GDPR-deletion users still exposed via certain admin endpoints.
**M-05 — No array-length validation on profile arrays** _(adversarial)_ — submit 100k entries in `interests[]`, blow up the row.
**M-06 — `portfolioLinks` accepts arbitrary strings** _(adversarial)_ — `javascript:`, `file:`, candidates for SSRF if backend ever fetches them.

### LOW (3)

**L-01** — `/api/users/:id` returns 404 for non-existent (good — recon noise).
**L-02** — Caddyfile leaves `:80` open with `auto_https disable_redirects`.
**L-03** — `ALLOW_UNVERIFIED=true` in dev hard-bypasses `requireVerified`. Fail-fast in `NODE_ENV=production` like JWT_SECRET does.
